📊 数据流总结
● PreLLMCall 聚合返回：memoryContext ragContext providerOptions → buildContextMessage + streamText
● 核心节点不持有业务逻辑：executeStream() 52 行纯编排，所有功能在 hook handler 中
● 副作用 handler：notification-hooks（session + taskRegistry）、turn-hooks（DB 写入）、compression-hooks（消息替换）
● 可阻断点：UserPromptSubmit、PreToolUse — handler 返回 {blocked:true} 即停止
