# docs/archive — 历史实施记录

本目录保存已经完成、合并、终止或被替代的 effort，包括 issue、design、plan、acceptance 和验收证据。

这些文件是历史记录：

- 保留当时的名称、路径、表结构和决策，通常不会随当前代码重写。
- 不能用来证明某个模块今天仍存在或某项验收今天仍通过。
- 查当前行为请看 [`../basic/`](../basic/README.md) 和 [`../arch/`](../arch/README.md)。
- 若历史文档与当前源码冲突，应更新当前文档或新建 issue，不要篡改已完成验收的原始叙事。

目录中可能同时存在成功方案、废止方案和中间 review；阅读具体 effort 时先看其 `issue.md`、`design.md`、README 或最终 acceptance，再结合 git 历史判断结果。
