# 当前文档基线

本目录描述已经落地的当前行为。最后一次代码与测试核对日期：2026-07-19。

## 事实优先级

遇到冲突时，按以下顺序判断：

1. 可执行测试与当前源代码。
2. `package.json`、TypeScript 配置和构建脚本。
3. 本目录的当前基线文档。
4. `docs/arch/` 的深度分析。
5. `docs/design/`、`docs/plan/` 的目标方案。
6. `docs/archive/` 的历史记录。

`docs/plan/` 中出现的类型、表、路由或行为只有在代码中存在并有测试覆盖时，才算已经实现。归档文档保留当时的决策与验收证据，不会随当前代码持续更新。

## 维护约定

- 避免把易漂移的文件数量、行号和通道总数写成架构事实。
- 命令必须来自当前 `package.json`；不存在的命令不得作为使用说明。
- 当前实现和未来设计必须分开表达。
- 文档修改后运行 `npm run check:links`；涉及行为描述时同时运行对应测试或类型检查。
