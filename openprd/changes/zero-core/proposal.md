# zero-core

## 背景与原因

开发者缺少可见性：不知道智能体使用哪些工具容易出错、什么上下文下出错、错误原因是什么，无法针对性地优化工具代码和提示词。

## 变更内容

- 数据库：新增 tool_executions 表记录工具执行详情
- 记录层：在 agent-loop 捕获工具调用和错误
- 查询层：IPC 接口支持查询统计数据
- 前端：ToolsPage 新增统计 tab
- 分析：用户选择工具后，AI 基于错误上下文生成建议
- FR1: 系统记录每次工具执行（工具名、参数、结果/错误、执行时间、关联的 session/turn）
- FR2: 前端展示工具错误统计（总调用数、错误数、错误率）
- FR3: 用户可选择工具进行分析
- FR4: AI 基于错误上下文生成分析报告
- FR5: 提供数据清理功能
- 数据库正确记录工具执行详情
- 前端正确展示错误统计
- AI 分析建议质量合格（给出具体原因和改进方向）

## 能力范围

- `consumer-requirements`: zero-core 需求。

## 影响范围

- 主要用户: zero-core 项目开发者
- 成本来源: LLM 分析调用产生 API 成本
- 额度限制: 每次分析消耗约 1k tokens
- 依赖: agent-loop 工具事件流
- 依赖: session-db 数据库访问
- 依赖: LLM provider
- 风险: AI 建议质量不可控
- 风险: 数据库写入可能影响工具执行性能
