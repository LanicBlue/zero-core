## 新增需求

### 需求：FR1 工具执行记录
系统在运行时循环中捕获每次工具调用，异步写入工具执行记录表，记录工具名、参数、结果或错误、执行时间、关联的会话和轮次。

#### 场景：工具调用成功被记录
- **当** 智能体执行工具成功
- **则** 工具名、参数、结果、执行时间被写入数据库，状态为成功

#### 场景：工具调用失败被记录
- **当** 智能体执行工具失败
- **则** 工具名、参数、错误消息、错误类型、执行时间被写入数据库，状态为失败

### 需求：FR2 工具错误统计展示
工具页面工具详情新增"统计"标签页，展示该工具的总调用数、错误数、错误率，以及最近错误列表（最多十条）。

#### 场景：开发者查看工具统计
- **当** 开发者选择工具并切换到统计标签页
- **则** 看到总调用数、错误数、错误率的统计卡片和最近错误列表

### 需求：FR3 工具分析选择
用户在统计标签页中可选择某个工具进行智能分析。

#### 场景：选择工具进行分析
- **当** 用户点击"智能分析建议"按钮
- **则** 系统调用语言模型基于该工具的错误记录生成分析

### 需求：FR4 智能分析建议
系统调用语言模型，基于该工具的错误记录分析原因，生成问题原因和改进方向的可折叠展示。

#### 场景：分析成功
- **当** 语言模型返回分析结果
- **则** 在页面内可折叠展示问题原因和改进方向

#### 场景：分析失败
- **当** 语言模型分析超时或返回错误
- **则** 显示分析失败错误信息

### 需求：FR5 数据清理
提供按时间清理旧工具执行记录的功能。

#### 场景：清理旧记录
- **当** 系统调用清理接口并传入最大保留时间
- **则** 超过保留时间的记录被删除，返回删除数量

#### 场景：工具无错误记录
- **当** 工具没有错误记录
- **则** 统计标签页显示"暂无数据"

#### 场景：错误记录不足
- **当** 错误记录少于五条
- **则** 提示"数据不足，建议积累更多数据"
